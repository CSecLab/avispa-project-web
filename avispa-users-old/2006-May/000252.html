Hello Franziska,

> /\_{in(A.B.Ka.Kb, Instances)}

This is one way of specifying the sessions to be analyzed.  This  
should be read as the parallel composition over all A.B.Ka.Kb in the  
set Instances, where Instances is a set of type  
(agent.agent.public_key.public_key) set.  What follows is probably  
something along the lines of
    Alice(A,Ka,...)
/\ Bob(B,Kb,...)

So this says, for each pair of agents and public keys in the set  
Instances, instantiate one Alice role and one Bob role.

This is not really deprecated, but it is rarely used these days.

In general, we tend to write specifications with one role called  
"session" which starts a single initiator, a single responder, and  
perhaps other roles if they are required for a full session of the  
protocol.  Then, rather than defining our analysis scenario by a set  
Instances, we simply instantiate one "session" role within the  
environment for each session we want to analyze.

I hope this helps.

Paul.
